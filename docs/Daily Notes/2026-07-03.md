## Work Done

**Deployed + verified the session 5 changes.**

- User hit two runtime errors after the frontend rebuilt but the backend image didn't: workspace-visibility toggles returned **422 "field required"**, and saving the Can-manage (pool_edit) permission returned **405 Method Not Allowed**.
- Root cause: **stale backend image** — the new `AdminSettingsRequest` (optional fields) and the new `PATCH /admin/users/{id}/pool-edit` route weren't loaded. The 405 comes from the SPA GET-only catch-all answering a PATCH to a not-yet-deployed path.
- Fix: rebuilt the backend image via `bash launch.sh --skip-build` (frontend dist was already current). `launch.sh` elevates Docker through `sg docker` automatically — no sudo password needed.
- Verified against the live OpenAPI schema: `/api/v1/auth/admin/users/{user_id}/pool-edit` is present, and `AdminSettingsRequest` now has `allow_open_registration` + `enabled_workspaces` with `required: none`. App healthy.

## Decisions Made

- Recorded the stale-backend symptom signature (405 on new routes, 422 "field required" on changed models) in MEMORY.md under the existing rebuild gotcha, so this is faster to diagnose next time.
