**Auth/theme (root of the sign-out mess):**
- `logout()` never reset the theme CSS vars → the signed-out user's background lingered on the login page until a reload (and could block the next login). Fixed: `logout()` resets background/accent/dark/density/corner.
- The 30s `/me` poll's `refreshUser()` `catch` called `setUser(null)` on ANY failure (transient blip) → random logout + "background reverts then returns" flicker. Fixed: catch is a no-op; genuine expiry is handled by `request('/auth/me')`'s 401 redirect.
- `e.filter` crash: client-side, couldn't be localized from the minified message (all `.filter` sites guarded) — the two auth fixes remove the churn that stuck the app in the ErrorBoundary. Added a global `window.onerror`/`unhandledrejection` console logger in `main.jsx`, kept ErrorBoundary stack logging, and added defensive `Array.isArray` guards (TagInput, AssetModal `allAssets`, Assets `load`). If it recurs the console stack now pinpoints it.

**Sharing model reworked (owner's "groups" idea):**
- Per-node resolution (dropped the ancestor walk in `_share_access`/`_is_hidden_from`). `update_access(cascade=True default)` writes the audience to the node + all descendants; `cascade=False` = this node only. UI toggle ("Apply to everything inside" / "This one only") shows only when the node has descendants.
- **Create-under-shared inheritance**: `create_asset` copies the parent's `shared_with`+`hidden_from` onto a new child; `POST /assets` routes a child into the **parent's store** (requires edit access). So an edit-shared user can grow a shared subtree owned by the original owner — a "group".
- Existing subtree tests stayed green because cascade defaults True (behavior preserved). 5 new tests; suite **261 green**; black/isort clean.

## Session — Assets Phase 2 (per-user templates + request-based share handshake)

Owner chose: build it all at once; **every share is a per-user request** (even group/role shares notify each member; nothing appears until accepted); decline/leave silently drops.

- **Actionable notifications**: `suggestions_service.add_notification(action=)` + `resolve_notification()`; NotifBell renders Accept/Decline → `POST /assets/shares/respond`.

Template sharing reuses the handshake; global restricted by role, can't be left.
