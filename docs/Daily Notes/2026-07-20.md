## Remediation pass 4 — residual account-security (account lockout)
Closed the account-lockout residual from the audit's outsider threat model.
- `auth_service.login_attempt()` + `_failed_logins` (email-keyed, in-memory):
  10 fails / 15 min → account temporarily refused (429) until failures age out.
  Sliding-window, never permanent; router checks lock BEFORE authenticate() so
  attempts-while-locked don't extend it (no victim-lockout DoS). Second layer
  under the per-IP limiter — stops distributed stuffing (many IPs, one account).
- `/auth/login` + `/auth/token` both route through `login_attempt()`.
- 6 new tests in test_auth_service.py. FULL SUITE GREEN: 467 passed; `main` imports OK.
- Docs: CHANGELOG Security, MEMORY rule 9a, audit doc Remediation pass 4.
