## Work Done

Fixed workspace mode switching bugs and added workspace awareness to Chat.

### Bug 1 вАФ Pages don't auto-refresh on workspace switch
- **Root cause**: Calendar, Tasks, Notes all had `useEffect` with only `[user?.name]` in deps вАФ no `workspace`
- **Fixed**: Added `useWorkspace` import + `workspace` to deps in Calendar.jsx and Tasks.jsx; added `workspace` to `useCallback` deps in Notes.jsx (triggers `load` ref change вЖТ `useEffect([load])` re-runs)

### Bug 2 вАФ No redirect when switching to a workspace where current module is restricted
- **Root cause**: Layout.jsx's `useEffect([workspace])` only called `refreshUser()` вАФ no check against the current page's workspace constraint
- **Fixed**: Added `useLocation` import; in the same effect, find the module matching `location.pathname` in `ALL_MODULES`; if `m.workspace && m.workspace !== workspace`, call `navigate('/')`. Covers JournalвЖТbusiness, HouseholdвЖТbusiness, TeamвЖТpersonal.

### Bug 3 вАФ Calendar shows Household UI in business mode
- **Root cause**: Calendar.jsx hardcoded `sharedApi` (household) regardless of workspace; EventModal had hardcoded "Household" labels
- **Fixed**:
  - Calendar.jsx: `poolApi = isPersonal ? sharedApi : teamApi`; state vars renamed `showHousehold` вЖТ `showPool` / `householdEvents` вЖТ `poolEvents` etc; toggle shows рЯП† or рЯІСвАНрЯ§ЭвАНрЯІС; passes `poolSaveApi` + `poolLabel` to EventModal
  - EventModal.jsx: `householdSaveApi` вЖТ `poolSaveApi`, added `poolLabel = 'Household'` prop; button text uses `poolLabel`
  - CalendarGrid.jsx: `ev._poolEmoji || 'рЯП†'` so business team events show рЯІСвАНрЯ§ЭвАНрЯІС badge

### Feature вАФ Chat workspace awareness + cross-workspace search toggle
- **chat.py**: `_build_context()` now takes `workspace` and uses `ws_path()` for memory + tasks. All chat endpoints (save, list, delete, save-memory) inject `workspace = Depends(get_workspace)` and use workspace-scoped Chats directory. `ChatRequest` adds `cross_workspace: bool = False`.
- **agent_service.py**: `run_agent()` accepts `workspace` + `cross_workspace`. `list_brain_files`, `read_brain_file`, `append_memory`, `rewrite_memory` use `ws_path()`. `search_brain` searches the active workspace by default; when `cross_workspace=True`, searches both personal and business paths (labels results with `personal/` or `business/` prefix).
- **api.js**: `chat.send()` now accepts `crossWorkspace` param, passes as `cross_workspace`.
- **Chat.jsx**: Added `useWorkspace`; `crossWorkspace` state; `useEffect([workspace])` resets toggle + reloads saved chats list; passes `crossWorkspace` to `chatApi.send()`; "This workspace"/"Both workspaces" toggle button shown only when `user.workspaces.length > 1`.

## Decisions Made

- `poolEmoji` is set at Calendar.jsx level when tagging events with `_household: true`, so CalendarGrid gets the right emoji without needing to know the workspace
- Chat's `Business/Chats/` folder is created on first save вАФ existing personal chats stay in `Chats/` (backward compat, no migration needed)
- Cross-workspace AI search only searches `.md` files (same skip list as single-workspace search); results are prefixed with `personal/` or `business/` so the AI knows which workspace each hit came from
- `read_brain_file` in agent_service now validates `.md` extension + no traversal, resolves against `ws_path()`, and boundary-checks against `user_path()` (both personal and business paths are within the user's brain root)

## Work Done (session 2)

Overhauled `docs_reminder.sh` and updated `docs_loader.sh` to make the end-of-turn docs reminder dynamic.

**docs_reminder.sh:**
- Now detects which project files changed since the last stop using `find -newer` against a temp reference file with the LAST_STOP timestamp (via `python3 os.utime`)
- Builds a targeted checklist of additional docs to update based on file patterns:
  - New untracked `app/` files вЖТ `docs/MAP.md`
  - Backend router changes вЖТ `docs/API.md`
  - `main.py` / `config.py` changes вЖТ `docs/MEMORY.md` (env vars / architecture)
  - `scheduler.py` changes вЖТ `docs/AGENTS.md` (Scheduler table)
  - New frontend pages вЖТ `docs/AGENTS.md` (Frontend Conventions вЖТ Pages)
  - Hook changes вЖТ `docs/AGENTS.md` (Dev Skills / hook workflow)
  - Services changes вЖТ `docs/MEMORY.md` (Core Design Decisions)
- Handles quoted paths from `git status -z` to avoid spaces-in-filenames breakage (was splitting "Daily Notes" as two tokens)
- Filters daily notes from MAP.md trigger (they don't affect the navigation index)

**docs_loader.sh:**
- When a pending reminder is present, adds a numbered step 2 to the REQUIRED block: "Address the PENDING DOCS UPDATE above вАФ update every listed doc that applies"
- Makes docs updates a first-class required action rather than just inline context

## Decisions Made

- `find -newer` with a python3-stamped reference file is the cleanest cross-minute mtime comparison in bash; avoids subshell timestamp juggling
- Exclude `docs/` untracked files from MAP.md trigger вАФ daily notes are the only typical untracked docs files and they don't affect MAP.md
- Filter `^_` prefixed router files from the API.md trigger (shared model files like `_task_models.py` don't define API routes)
