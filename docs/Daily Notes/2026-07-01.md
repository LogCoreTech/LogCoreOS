## Work Done

Fixed workspace mode switching bugs and added workspace awareness to Chat.

### Bug 1 — Pages don't auto-refresh on workspace switch
- **Root cause**: Calendar, Tasks, Notes all had `useEffect` with only `[user?.name]` in deps — no `workspace`
- **Fixed**: Added `useWorkspace` import + `workspace` to deps in Calendar.jsx and Tasks.jsx; added `workspace` to `useCallback` deps in Notes.jsx (triggers `load` ref change → `useEffect([load])` re-runs)

### Bug 2 — No redirect when switching to a workspace where current module is restricted
- **Root cause**: Layout.jsx's `useEffect([workspace])` only called `refreshUser()` — no check against the current page's workspace constraint
- **Fixed**: Added `useLocation` import; in the same effect, find the module matching `location.pathname` in `ALL_MODULES`; if `m.workspace && m.workspace !== workspace`, call `navigate('/')`. Covers Journal→business, Household→business, Team→personal.

### Bug 3 — Calendar shows Household UI in business mode
- **Root cause**: Calendar.jsx hardcoded `sharedApi` (household) regardless of workspace; EventModal had hardcoded "Household" labels
- **Fixed**:
  - Calendar.jsx: `poolApi = isPersonal ? sharedApi : teamApi`; state vars renamed `showHousehold` → `showPool` / `householdEvents` → `poolEvents` etc; toggle shows 🏠 or 🧑‍🤝‍🧑; passes `poolSaveApi` + `poolLabel` to EventModal
  - EventModal.jsx: `householdSaveApi` → `poolSaveApi`, added `poolLabel = 'Household'` prop; button text uses `poolLabel`
  - CalendarGrid.jsx: `ev._poolEmoji || '🏠'` so business team events show 🧑‍🤝‍🧑 badge

### Feature — Chat workspace awareness + cross-workspace search toggle
- **chat.py**: `_build_context()` now takes `workspace` and uses `ws_path()` for memory + tasks. All chat endpoints (save, list, delete, save-memory) inject `workspace = Depends(get_workspace)` and use workspace-scoped Chats directory. `ChatRequest` adds `cross_workspace: bool = False`.
- **agent_service.py**: `run_agent()` accepts `workspace` + `cross_workspace`. `list_brain_files`, `read_brain_file`, `append_memory`, `rewrite_memory` use `ws_path()`. `search_brain` searches the active workspace by default; when `cross_workspace=True`, searches both personal and business paths (labels results with `personal/` or `business/` prefix).
- **api.js**: `chat.send()` now accepts `crossWorkspace` param, passes as `cross_workspace`.
- **Chat.jsx**: Added `useWorkspace`; `crossWorkspace` state; `useEffect([workspace])` resets toggle + reloads saved chats list; passes `crossWorkspace` to `chatApi.send()`; "This workspace"/"Both workspaces" toggle button shown only when `user.workspaces.length > 1`.

## Decisions Made

- `poolEmoji` is set at Calendar.jsx level when tagging events with `_household: true`, so CalendarGrid gets the right emoji without needing to know the workspace
- Chat's `Business/Chats/` folder is created on first save — existing personal chats stay in `Chats/` (backward compat, no migration needed)
- Cross-workspace AI search only searches `.md` files (same skip list as single-workspace search); results are prefixed with `personal/` or `business/` so the AI knows which workspace each hit came from
- `read_brain_file` in agent_service now validates `.md` extension + no traversal, resolves against `ws_path()`, and boundary-checks against `user_path()` (both personal and business paths are within the user's brain root)

## Work Done (session 5)

Fixed workspace-switch redirect not firing for business-only modules (Team, etc.).

**Bug:** `Layout.jsx` `useEffect([workspace])` used `location.pathname.startsWith(m.to)` to find the current module. `dashboard` has `to: '/'`, and every pathname starts with `/`, so `find()` always returned `dashboard` first — which has no `workspace` field — causing the condition to short-circuit and `navigate('/')` to never fire.

**Fix:** [Layout.jsx:138](app/frontend/src/components/Layout.jsx#L138) — treat `'/'` as exact match: `m.to === '/' ? location.pathname === '/' : location.pathname.startsWith(m.to)`.
