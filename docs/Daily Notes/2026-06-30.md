# 2026-06-30 ‚Äî Daily Note

## Work Done

Fixed 5 bugs in the personal/business workspace and feature roles system introduced in PR #4.

**Bug 1 ‚Äî Role creation returned [object Object]:**
- Root cause: `goals` in frontend `ALL_MODULES` but not in backend `ALL_MODULE_IDS` ‚Üí 422 with array `detail` ‚Üí `new Error(array)` ‚Üí `[object Object]`
- Fixed: added `goals` to `ALL_MODULE_IDS` and both default role maps in `features_service.py`
- Fixed: `api.js request()` now extracts `.msg` fields from array `detail` responses (handles all Pydantic validation errors)

**Bug 2 ‚Äî Module restrictions not applied on login / workspace switch lag:**
- Root cause A: `/login` returned raw `user.get("disabled_modules", [])` instead of calling `get_effective_disabled()`
- Root cause B: `Login.jsx` didn't pass `me.workspaces` to `auth.login()`, so `user.workspaces` was undefined until the first `/me` poll
- Root cause C: switching workspace didn't re-fetch `/me`, so `disabledModules` could be 30s stale
- Fixed: `/login` and `/register` now call `get_effective_disabled()` and return `workspaces`; `auth.jsx` accepts `workspaces` in `login()` and exposes `refreshUser()`; Layout.jsx calls `refreshUser()` on workspace change

**Bug 3 ‚Äî Automations network error when n8n unconfigured:**
- Added `n8n_service.is_configured()` (checks `api_key` non-empty)
- Added guard to `run`, `activate`, `deactivate`, `logs` endpoints ‚Üí 503 with clear message

**Bug 4 ‚Äî Admin didn't see workspace switcher:**
- `/register` now gives first user (admin) `["personal", "business"]` at registration
- `get_current_user()` lazy-migrates existing admins: writes `["personal", "business"]` on their first authenticated request if business is missing

**Bug 5 ‚Äî Workspace didn't filter irrelevant modules:**
- Added `workspace` field to `ALL_MODULES` in `constants.js`: `household`, `journal`, `home` ‚Üí `personal`; `team`, `automations_business` ‚Üí `business`
- Layout.jsx `visibleModules` filter respects `m.workspace`
- Automations.jsx Business tab requires `workspace === 'business'`
- Mobile drawer gained workspace switcher (was desktop-sidebar only)

## Decisions Made

- `goals` is a frontend-only page (filters tasks by type) ‚Äî adding it to `ALL_MODULE_IDS` makes it independently gateable by role without needing a separate backend router guard
- Workspace filtering is frontend-only (UX concern); backend `require_module()` still enforces access regardless of workspace
- `disabled_modules` is computed at login time to avoid the brief window where restrictions appear missing
- Workspace auto-switch: if active workspace isn't in user's allowed list, Layout auto-switches to `userWorkspaces[0]`

## Follow-up

- None ‚Äî roles, workspace switching, and module visibility are now stable
