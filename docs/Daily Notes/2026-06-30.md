# 2026-06-30 — Daily Note

## Work Done

Fixed 5 bugs in the personal/business workspace and feature roles system introduced in PR #4.

**Bug 1 — Role creation returned [object Object]:**
- Root cause: `goals` in frontend `ALL_MODULES` but not in backend `ALL_MODULE_IDS` → 422 with array `detail` → `new Error(array)` → `[object Object]`
- Fixed: added `goals` to `ALL_MODULE_IDS` and both default role maps in `features_service.py`
- Fixed: `api.js request()` now extracts `.msg` fields from array `detail` responses (handles all Pydantic validation errors)

**Bug 2 — Module restrictions not applied on login / workspace switch lag:**
- Root cause A: `/login` returned raw `user.get("disabled_modules", [])` instead of calling `get_effective_disabled()`
- Root cause B: `Login.jsx` didn't pass `me.workspaces` to `auth.login()`, so `user.workspaces` was undefined until the first `/me` poll
- Root cause C: switching workspace didn't re-fetch `/me`, so `disabledModules` could be 30s stale
- Fixed: `/login` and `/register` now call `get_effective_disabled()` and return `workspaces`; `auth.jsx` accepts `workspaces` in `login()` and exposes `refreshUser()`; Layout.jsx calls `refreshUser()` on workspace change

**Bug 3 — Automations network error when n8n unconfigured:**
- Added `n8n_service.is_configured()` (checks `api_key` non-empty)
- Added guard to `run`, `activate`, `deactivate`, `logs` endpoints → 503 with clear message

**Bug 4 — Admin didn't see workspace switcher:**
- `/register` now gives first user (admin) `["personal", "business"]` at registration
- `get_current_user()` lazy-migrates existing admins: writes `["personal", "business"]` on their first authenticated request if business is missing

**Bug 5 — Workspace didn't filter irrelevant modules:**
- Added `workspace` field to `ALL_MODULES` in `constants.js`: `household`, `journal`, `home` → `personal`; `team`, `automations_business` → `business`
- Layout.jsx `visibleModules` filter respects `m.workspace`
- Automations.jsx Business tab requires `workspace === 'business'`
- Mobile drawer gained workspace switcher (was desktop-sidebar only)

## Decisions Made

- `goals` is a frontend-only page (filters tasks by type) — adding it to `ALL_MODULE_IDS` makes it independently gateable by role without needing a separate backend router guard
- Workspace filtering is frontend-only (UX concern); backend `require_module()` still enforces access regardless of workspace
- `disabled_modules` is computed at login time to avoid the brief window where restrictions appear missing
- Workspace auto-switch: if active workspace isn't in user's allowed list, Layout auto-switches to `userWorkspaces[0]`

## Follow-up

- None — roles, workspace switching, and module visibility are now stable

## Work Done (session 2)

Fixed shortcuts settings and added per-workspace shortcut preferences.

**Bug — shortcuts picker showed disabled/wrong-workspace modules:**
- Root cause: `ALL_MODULES.filter(m => !shortcutIds.includes(m.id))` had no disabled or workspace filter
- Fixed: picker now filters `!user?.disabledModules?.includes(m.id)` and `(!m.workspace || m.workspace === ws)`
- Also fixed pinned list to skip rendering disabled/wrong-workspace modules

**Feature — independent shortcut preferences per workspace:**
- Replaced single `shortcutIds` state with `personalShortcutIds` + `businessShortcutIds`
- Settings card shows a Personal panel always; Business panel only if `user.workspaces.includes('business')`
- Each panel has its own pinned list, add picker, and drag-to-reorder — filtered to that workspace's valid modules
- Both workspaces saved in one `PATCH /auth/me` call

**Bug — fourth shortcut slot blocked by hidden invalid IDs:**
- Root cause: `cleanShortcuts()` was filtering the display but the `ids` array still held the invalid ID, blocking the slot
- Fixed: `cleanShortcuts(ws, user)` strips disabled and wrong-workspace IDs at init and on server sync so `ids.length` is always the true usable count

## Decisions Made

- `cleanShortcuts()` runs at init and on `user?.shortcuts` change — not on `user?.disabledModules` change (acceptable: shortcuts reset on next sync or re-login)
- Business panel is hidden entirely (not just greyed) for personal-only users to avoid confusing dead UI
