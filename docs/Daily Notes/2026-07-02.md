- Frontend: `user.poolEdit` from `/me` (`auth.jsx`); `canEdit`/`canEditPool` in Household/Team/Calendar use it; Household/Team fetch members via the new endpoints; Admin UsersCard "Can manage" row (Household/Team checkboxes) → `adminApi.updatePoolEdit`.

## Decisions Made (session 5)

- `pool_edit` is a dedicated user field, **not** a module in `disabled_modules` — the union model can't express grants (only restrictions). Documented in MEMORY.md.
- Workspace hiding is enforced server-side (intersection in `get_current_user`) so the pill, admin assignment, and data access all follow from one place; persisted per-user workspaces are left intact so re-enabling restores access.
- Pool grant gives **full admin parity** on the pool (per user's choice), including event edit/delete which was previously admin-only.
